Posicionamiento CSS

La propiedad position permite modificar la posicion natural de cualquier elemento del documento HTML debido a que el navegador coloca los elementos en el orden en el que van apareciendo

Tipos de posicionamiento

  • Static (por defecto)
  • Relative
  • Absolute
  • Fixed
  • Sticky

Para mover los elementos posicionados se activan las propiedades: (no aplica para el valor de static)

  • Top (eje y)
  • Bottom (eje y)
  • Left (eje x)
  • Right (eje x)
  • z-index (eje z)

Las propiedades top y left tienen prioridad sobre bottom y right

Caja estatica. Es el valor por defecto y respeta el flujo del documento HTML por lo que no se puede mover ni posicionar
Caja relativa. El posicionamiento relative conserva sus dimensiones (alto y ancho) y posicion original (orden que tiene en el HTML) en el flujo del documento. Si se mueve toma como referencia su posicion original
Caja absoluta. El elemento pierde sus dimensiones y posicion original en el flujo del documento HTML. Si se mueve puede tomar como referencia dos elementos
  1. El primer ancestro con posicionamiento relativo
  2. Sino encuentra un ancestro relativo se mueve con respecto al elemento HTML
Caja absoluta con ancestro relativo
Caja fija. El elemento pierde sus dimensiones y posicion original en el flujo del documento. Si se mueve toma como referencia el documento HTML y queda fijo en la posicion. Cuando se haga scroll en el documento el elemento quedara fijo en esa misma posicion
Caja pegajosa. Es una combinación de posicionaiento relative y fixed, las propiedades de top, botton... sirven como punto de referencia. Hasta que el scroll no llegue a los valores puestos en las propiedades el elemento se va a comportar como un elemento relative, pero apenas llega se comporta como un fixed. El elemento debe ser hijo directo del body, si tiene un elemento padre solo será sticky dentro de las dimensiones (si o si debe definirle el ancho y alto) del contendedor padre.
Caja sticky como hijo

Z-index

z-index es una propiedad que permite colocarle profundida a los elementos posicionados. Acepta numeros positivos, negativos y 0

A mayor valor, el elemento estará más al frente, a menos valor más al fondo. Un hijo no podrá estar detras de un padre a menos que se le de un valor negativo y que el elemento padre no tenga un valor de z-index
